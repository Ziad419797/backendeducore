طبقات كل موديول (3-Layer Architecture)
🌐
Controller Layer
REST API — Presentation
@RestController بيستقبل الـ HTTP requests
@RequestMapping بيحدد الـ base URL لكل resource
ResponseEntity بيرجع الـ response مع الـ status code
@Valid / @Validated validation على الـ request body
⚙️
Service Layer
Business Logic
@Service + @Transactional كل العمليات في transaction
@Transactional(readOnly=true) على الـ GET methods للأداء
ResourceNotFoundException custom exception لو مش موجود
SortValidator بيتحقق من صحة الـ sort fields
🗄️
Repository Layer
Data Access — JPA
JpaRepository CRUD + Pagination جاهزة
@Query (JPQL) queries معقدة بـ JOIN FETCH
Pageable pagination + sorting تلقائي
existsById / existsByTitle فحص الوجود بدون تحميل كامل
🔄
Mapper Layer
DTO ↔ Entity (MapStruct)
toEntity(request) DTO → Entity عند الإنشاء
toResponse(entity) Entity → DTO عند الإرجاع
updateEntityFromRequest partial update بدون null overwrite
toSummaryResponse نسخة خفيفة للـ lists

جداول العلاقات في الـ Database
الـ Owner (صاحب العلاقة) نوع العلاقة الطرف التاني اسم الجدول في الـ DB الـ Columns
Course ⭐ ManyToMany Category course_category course_id, category_id
Course ⭐ ManyToMany Session course_sessions course_id, session_id
Week ⭐ ManyToMany Session session_weeks week_id, session_id
Week ⭐ ManyToMany LessonMaterial week_materials week_id, material_id
Level OneToMany Category Categories (FK) level_id في جدول Category

⭐ = الـ Owner هو اللي بيتحفظ العلاقة في الـ DB — لازم تضيف/تحذف العلاقة دايماً منه

الإصلاحات المطلوبة
CRITICAL — تم الإصلاح ✅
تناقض الـ JoinTable
كان Session بيعرّف session_course وCourse بيعرّف course_sessions — خلق جدولين في الـ DB بدل جدول واحد.

الحل: Course هو الـ owner الوحيد بجدول course_sessions، و Session بيستخدم mappedBy فقط.
CRITICAL — تم الإصلاح ✅
@CreationTimestamp على updatedAt
في Category وSession، الـ updatedAt كان بيستخدم @CreationTimestamp بدل @UpdateTimestamp.

النتيجة: الحقل كان بيتحفظ مرة واحدة ومش بيتحدث أبداً — بيانات خاطئة في الـ DB.
MEDIUM — مطلوب لاحقاً
N+1 Query Problem
الـ findById بيجيب الـ entity بدون relations، وبعدين كل relation بتتحمل بـ query منفصلة.

الحل: إضافة JOIN FETCH في الـ Repository للـ queries الأساسية.
MEDIUM — مطلوب لاحقاً
أسماء متناقضة
LessonService بتتكلم عن Week — وSeccionRepository بدل SessionRepository — والجدول اسمه weeks بس الـ Entity اسمها Session.

الحل: توحيد الأسماء قبل الـ production.
